Definition

What is Symbia Seed?

Symbia Seed is a local-first execution runtime for running autonomous or semi-autonomous systems under explicit governance. It is not a model, and it is not a prompt framework. It sits above models and protocols and decides what work is allowed to run, under which identity, and with what constraints.

Boundaries

What Seed is not

Not a large language model.

Not an agent marketplace.

Not a workflow SaaS.

Not a replacement for MCP or APIs. Models generate predictions. Protocols move messages. Seed governs execution.

Why

Why Seed exists

When AI systems move from demos into real environments, the missing pieces are always the same: identity, continuity, constraints, and auditability. Seed makes those first-class instead of bolting them on with prompts and logs.

Architecture

How Seed works (conceptually)

Canonical identity rooted in a genesis key.

Derived keys for installs, sessions, actors, and workers.

A mission and worker model for bounded execution.

An abstraction layer that records meaningful state transitions.

Artifact boundaries that enforce epistemic and policy constraints.

MCP, HTTP, MQTT, and other protocols are treated as ingress and egress. They never define cognition or authority.

Quickstart

Quickstart: run Seed locally

Seed is designed to be runnable on your own machine. No hosted service is required.

1. Download

Download the latest Symbia Seed release or clone the repository:

git clone https://github.com/Symbia-Labs/symbia-seed
cd symbia-seed

2. Install dependencies

Follow the installation instructions for your platform:

./seed.sh install

3. Boot a seed

Start a local Seed instance:

./seed.sh boot

4. Run your first mission

Create and execute a mission:

./seed.sh mission run examples/hello-world.yaml

These commands are intentionally simple. The complexity lives inside the runtime, not in user glue code.

Supporting components

Supporting components

Genesis Key System (GKS)

An open specification for canonical, non-inspectable identity. Seed uses GKS to root all execution in a durable identity.

Open Epistemic Protocol (OEP)

An open contract for expressing knowledge, inference, and uncertainty at artifact boundaries. Seed enforces OEP at runtime.

Relationship

Research, specs, and runtime

Symbia Labs publishes research and open specifications. Symbia Seed is a governed runtime that embodies those ideas. The specs are portable; the runtime is opinionated by design.
