Alpha-2 updates

Transparent release notes

What users reported and how we shipped the fix.

Missionctl pytest detection

  • Bug: Script reported "pytest not found" even when a virtualenv was active.
  • Fix: Auto-bootstraps pytest into .venv if missing and limits discovery to Alpha-2 tests so no extra deps are required.

Installer temp path (macOS)

  • Bug: Mixed /tmp locations could leave archives and checksums in different dirs.
  • Fix: Single mktemp -d temp dir on macOS/Linux, logged paths, cleanup on exit.

Checksum determinism

  • Bug: Verification ran in the current dir; mismatched filenames could sneak through.
  • Fix: Hash the exact downloaded archive and compare to the expected SHA256 from the release.

Executive summary

What Alpha-2 is

A guided, auditable mission proving the runtime can say ‚Äúno‚Äù reliably, say ‚Äúyes‚Äù narrowly, and leave receipts either way.

Stable install identity and origin time.

Authenticated actor surfaces; default-deny privileged tools.

Scoped authorization per capability.

Governed network egress (private IPs blocked, allowlisted hosts accepted).

Durable records (logs + reflection stream) for audit.

Continuity semantics: state pointer, stable identity across calls.

Out of scope (by design)

  • No claim about model correctness or general safety.
  • Not a full isolation or key-hardening proof.
  • Does not prove queued requests were executed‚Äîonly accepted/denied and recorded.
  • No performance, load, or fuzz testing.

Threat model

What these tests defend against

Unauthenticated tool execution.

Privilege escalation via spoofed actors/headers.

Silent side effects (no durable records).

Network exfil / SSRF to private/internal targets.

State ambiguity (no stable install identity).

No receipts (missing trace/queue evidence).

Not a claim to defeat a fully compromised host; establishes minimum governable controls.

Quick start

Install, verify, run

Install (no git clone)

curl -fsSL https://raw.githubusercontent.com/Symbia-Labs/symbia-alpha2-early-access/main/install-alpha2.sh | bash

Installs to ~/.symbia-seed-dist; shims in ~/bin (symbia, alpha-2).

Integrity checks

  • SHA256 required; installer aborts if missing/mismatched.
  • Tarballs cosign-signed (GitHub OIDC).
cosign verify-blob --certificate <tar.gz>.crt --signature <tar.gz>.sig \
certificate-identity-regexp "https://github.com/Symbia-Labs/symbia-seed/.*" \
certificate-oidc-issuer https://token.actions.githubusercontent.com <tar.gz>

Run commands

  • symbia boot dev
  • alpha-2 missionctl (15s wait for endpoints)
  • Run steps 1‚Üí7 in order (Step 6 expects prior steps).

Notes

  • missionctl waits for API healthy before steps.
  • State dir: ~/.symbia-seed-alpha2; pointer: ~/.symbia-seed-current
  • Logs: ${STATE_DIR}/logs/symbia-seed.api.log
  • Traces: ${STATE_DIR}/logs/reflection/events.ndjson
  • Demo token: ${STATE_DIR}/auth/demo.token (chmod 600)

Test suite

Alpha-2 missionctl steps (1‚Äì7)

Step 1 ‚Äî Boot + Identity

Call /health and /status; expect status 200, version alpha2-‚Ä¶, install.id and t0_iso present.

Reachable runtime with stable install identity.

Step 2 ‚Äî Filesystem default-deny

POST /actors/fs without auth ‚Üí 401/403; no queue entry.

Blocks unauth privileged tool invocation.

Step 3 ‚Äî Filesystem allowlist

Mint dev token for actor.fs.read; list state dir; expect 200 queued; fs-requests.ndjson entry.

Scoped authz + queue receipts.

Step 4 ‚Äî Network default-deny

POST /actors/http url=http://127.0.0.1 denied (private IP block).

Egress governance / SSRF prevention.

Step 5 ‚Äî Network allowlist

POST /actors/http allowlisted host accepted and queued; http-requests.ndjson entry.

Only allowlisted outbound, recorded as receipts.

Step 6 ‚Äî Trace continuity

GET /reflection/events?limit=5 returns events; reflection log exists.

Evidence stream beyond terminal output.

Step 7 ‚Äî Restart continuity

~/.symbia-seed-current present; two /status calls return same install.id + t0_iso.

Stable identity and continuity pointer.

Audit posture

Cybersecurity, privacy, reliability

Cybersecurity

Auth boundary enforced; scoped authz; default-deny + governed egress; allowlisting; audit/event trails; stable identity.

Privacy

No data access without auth; narrow grants; no arbitrary outbound; accountability via logs; clear continuity boundaries.

Reliability

Readiness/status endpoints; fail-closed; queue acceptance; reflection events; state pointer + stable identity.

Scope boundary

Alpha-2 only

Included: Boot + identity; FS default-deny; FS allowlist; HTTP default-deny (private IP block); HTTP allowlist (queue only); trace continuity; restart continuity.

Excluded: any additional tests/extensions or future-phase assertions.
