Symbia

What is Symbia?

Symbia is an execution-layer doctrine for agentic systems.

It governs how software agents act, not how models predict.

Protocols connect components. Models produce probabilities.

Symbia defines what is allowed to run, where, and under whose responsibility.

Boundaries

What Symbia is not

Not a model.

Not a chatbot.

Not a protocol.

Not an autonomous system that replaces human responsibility.

Origin

Why Symbia

Automation and autonomy without governance are unsafe.

Symbia keeps control, accountability, and visibility intact.

The gap

Ungoverned execution is risk.

Actions run without boundaries. Uncertainty hides. Responsibility blurs.

Autonomy without enforcement creates liability.

Governed execution

  • Uncertainty is declared and carried forward.
  • Surfaces, time, and authority stay scoped.

Run

From canon to artifact

  • Start Seed: ./scripts/seed.sh start (API 8123, Observer 9000).
  • POST /event or /trace to log missions and worker activity.
  • GET /log/stream to watch canon move toward artifacts.
  • View docs and status without leaving the runtime.

OEP

Epistemics with guardrails

Envelope keeps obligations/refusals/unknowns explicit.

obligations: cite_sources, mark_hypotheses
refusals: missing_provenance, user_constraints_violation
epistemic_types: claim, hypothesis, observation, refusal
envelope: sha256 evidence hash, lineage: genesis_key_fingerprint

Identity

Deterministic lineage

Genesis Key fingerprints and Symbia Key constraints show up in status, identity snapshots, and traces. Evidence stays probabilistic; lineage stays deterministic.
